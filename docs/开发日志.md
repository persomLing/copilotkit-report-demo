- 分析项目的方向：用 React 页面状态和动作注册，让 AI 控制已有业务 UI。

- 明确 CopilotKit 更适合旧 React 项目渐进改造。

### 8. 导出状态问题修复

问题：

- AI 选中当前页后立即导出，导出工具读到旧的 `selectedIds`。

修复：

- 增加 `reportsRef`。
- 增加 `selectedIdsRef`。
- 让工具 handler 读取 ref。
- 手动勾选、AI 选中、查询刷新都同步 ref。

### 9. 当前页导出语义优化

问题：

- 用户说“导出当前页面数据”，AI 可能先调用选择，再导出选中项。

修复：

- 新增 `exportVisibleReports`。
- 当前页导出不需要先选中。
- 选中项导出和当前页导出拆成两个工具。

### 10. 二级筛选语义路由

需求：

- 用户直接说“可转债”等二级词时，AI 不能直接设置二级筛选。
- AI 需要有金融背景，但真实二级数据要通过接口获取。

实现：

- 增加 `SECONDARY_TERM_HINTS` 金融相似词典。
- 增加 `resolveSecondaryTermHint`。
- 增加 `resolveSecondaryFilterIntent` 工具。
- 页面上下文不再直接暴露 `secondaryOptions` 数组。
- `setReportFilter` 增加 `secondaryConfirmedByUser` 硬校验。

### 11. 接口等待和重试

需求：

- 接口请求有耗时。
- AI 应等待接口结果。
- 接口失败后自动重试一次。
- 重试仍失败时告知用户。

实现：

- 增加 `requestSecondaryOptions`。
- `fetchSecondaryOptions` 统一处理等待、重试、失败。
- `loadSecondaryFilterOptions` 返回结构化结果：
  - `ok`
  - `attempts`
  - `error`
  - `nextStep`

### 15. AG-UI 软打断与共享操作日志

本次根据业务语义重新定义“打断”：

- 用户打断只停止 AI 当前 run 的继续输出和后续工具规划。
- 已经发出的前端业务请求不再取消。
- 二级筛选接口、研报加载等请求完成后仍然写入共享页面状态。
- 这样更接近真实 B 端系统：进入后端/接口执行阶段的动作默认继续完成，避免半路取消导致业务状态不确定。

实现调整：

- 保留 `agent.abortRun()`。
- 移除前端业务请求对 `AbortController` 的依赖。
- 将运行中提示从“请求可取消”调整为“AI 后续操作可打断，已发请求继续完成”。
- 后端 agent prompt 同步更新，避免 AI 误以为业务请求被取消。

### 16. 结构化 operationLog

新增结构化操作日志，用于支撑用户追问：

- “我刚才做了什么？”
- “AI 刚才做了什么？”
- “现在页面是怎么变成这样的？”

日志字段：

- `actor`: `user` / `ai` / `system`
- `type`: 动作类型，例如 `setFilterField`、`loadReports`、`openReportDetail`
- `status`: `running` / `completed` / `failed` / `interrupted`
- `summary`: 给人看的摘要
- `payload`: 结构化参数和结果
- `createdAt`: 时间

接入范围：

- 用户手动筛选、加载、清空、导出、翻页、排序、勾选、打开详情。
- AI 设置筛选、加载列表、打开详情、选择研报、导出。
- 系统接口等待、成功、失败、重试。

Copilot 上下文新增 `recentOperations`，AI 回答历史操作问题时必须优先基于该字段，而不是依赖对话记忆猜测。
