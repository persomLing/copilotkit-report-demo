# CopilotKit 研报工作台 Demo 开发日志

## 2026-07-01

### 1. 项目分析

- 分析 `yingmi_dialogue_agent` 项目的方向：用 React 页面状态和动作注册，让 AI 控制已有业务 UI。
- 判断 Demo 本质是“状态派”：AI 不直接点 DOM，而是通过状态、工具和业务 action 改变页面。
- 明确 CopilotKit 更适合旧 React 项目渐进改造。

### 2. CopilotKit Demo 初始化

- 创建 `copilotkit-report-demo`。
- 使用 Vite + React。
- 引入：
  - `@copilotkit/react-core`
  - `@copilotkit/runtime`
  - `@ai-sdk/openai`
  - `express`
  - `zod`
- 建立前端工作台页面和后端 Runtime。

### 3. 模型接入

- 接入 DeepSeek OpenAI-compatible endpoint。
- 使用 `.env` 存储后端密钥。

### 8. 导出状态问题修复

问题：

- AI 选中当前页后立即导出，导出工具读到旧的 `selectedIds`。

修复：

- 增加 `reportsRef`。
- 增加 `selectedIdsRef`。
- 让工具 handler 读取 ref。
- 手动勾选、AI 选中、查询刷新都同步 ref。

### 9. 当前页导出语义优化

问题：

- 用户说“导出当前页面数据”，AI 可能先调用选择，再导出选中项。

修复：

- 新增 `exportVisibleReports`。
- 当前页导出不需要先选中。
- 选中项导出和当前页导出拆成两个工具。

### 10. 二级筛选语义路由

需求：

- 用户直接说“可转债”等二级词时，AI 不能直接设置二级筛选。
- AI 需要有金融背景，但真实二级数据要通过接口获取。

实现：

- 增加 `SECONDARY_TERM_HINTS` 金融相似词典。
- 增加 `resolveSecondaryTermHint`。
- 增加 `resolveSecondaryFilterIntent` 工具。
- 页面上下文不再直接暴露 `secondaryOptions` 数组。
- `setReportFilter` 增加 `secondaryConfirmedByUser` 硬校验。

### 11. 接口等待和重试

需求：

- 接口请求有耗时。
- AI 应等待接口结果。
- 接口失败后自动重试一次。
- 重试仍失败时告知用户。

实现：

- 增加 `requestSecondaryOptions`。
- `fetchSecondaryOptions` 统一处理等待、重试、失败。
- `loadSecondaryFilterOptions` 返回结构化结果：
  - `ok`
  - `attempts`
  - `error`
  - `nextStep`
