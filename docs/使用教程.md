# CopilotKit 研报工作台 Demo 使用教程

## 1. 项目说明

这是一个演示“AI 控制已有 React 业务页面”的 CopilotKit Demo。

核心目标：

- AI 理解当前页面状态。
- AI 调用前端注册的工具。
- 前端工具真正修改页面状态。
- 后端 Runtime 统一保存模型密钥和调用模型。

## 2. 环境要求

本地运行需要：

- Node.js 20 或更高版本，推荐 Node.js 22。
- npm。
- DeepSeek API Key 或 OpenAI API Key。

Docker 运行需要：

- Docker。
- Docker Compose。

## 3. 本地开发运行

进入项目目录：

```bash
cd D:\sy\2026\yingmi_dialogue_agent\copilotkit-report-demo
```

安装依赖：

```bash
npm install
```

复制环境变量文件：

```bash
copy .env.example .env
```

填写 `.env`：

```env
DEEPSEEK_API_KEY=sk-your-deepseek-key
PORT=4000
```

启动开发环境：

```bash
npm run dev
```

访问：

```text
http://localhost:5174
```

Runtime 地址：

```text
http://localhost:4000/api/copilotkit
```

健康检查：

```text
http://localhost:4000/health
```

## 4. Docker 运行

确保 `.env` 已存在，且包含模型密钥：

```env
DEEPSEEK_API_KEY=sk-your-deepseek-key
PORT=4000
```

构建并启动：

```bash
docker compose up --build
```

访问：

```text
http://localhost:4000
```

停止：

```bash
docker compose down
```

注意：

- Docker 镜像不会打包 `.env`。
- `.env` 通过 `docker-compose.yml` 的 `env_file` 注入。
- 不要把真实 `.env` 发给别人，只发 `.env.example`。

## 5. 推荐体验流程

### 5.1 普通筛选

输入：

```text
只看中信证券的报告，按评级降序排列
```

预期：

- AI 设置券商。
- AI 设置排序。
- AI 加载列表。

### 5.2 一级筛选

输入：

```text
看债券研究的报告
```

预期：

- AI 设置一级分类为债券研究。
- AI 加载列表。

### 5.3 疑似二级筛选

输入：

```text
看可转债的报告
```

预期：

- AI 不应直接筛选。
- AI 先调用 `resolveSecondaryFilterIntent`。
- AI 追问是否按“债券研究下的可转债方向”筛选。

用户确认：

```text
确认
```

预期：

- AI 调用 `loadSecondaryFilterOptions`。
- AI 等待二级候选项接口返回。
- AI 调用 `setReportFilter`，并带上 `secondaryConfirmedByUser=true`。
- AI 调用 `loadReports`。

### 5.4 当前页导出

输入：

```text
导出当前页面的数据
```

预期：

- AI 调用 `exportVisibleReports`。
- 浏览器下载当前表格 CSV。

### 5.5 选中项导出

输入：

```text
选中当前页所有报告，然后导出
```

预期：

- AI 调用 `selectReports`。
- AI 调用 `exportSelectedReports`。
- 前端弹出确认。
- 确认后下载 CSV。

## 6. 接口失败模拟

`loadSecondaryFilterOptions` 支持测试参数：

```text
simulateRequestCase = normal
simulateRequestCase = retrySuccess
simulateRequestCase = retryFail
```

含义：

- `normal`：接口正常返回。
- `retrySuccess`：第一次失败，第二次成功。
- `retryFail`：两次都失败。

真实对话中可以让 AI 模拟：

```text
模拟二级筛选接口第一次失败后重试成功，然后查可转债
```

或者：

```text
模拟二级筛选接口连续失败
```

预期：

- 第一次失败时自动重试。
- 两次失败时 AI 告知用户接口暂时不可用。
- 失败后不继续调用 `setReportFilter`。

## 7. 代码结构

```text
src/main.jsx
```

前端入口，注册 `CopilotKit` provider。

```text
src/App.jsx
```

页面主体、状态管理、Copilot 上下文、前端工具注册。

```text
src/reportStore.js
```

模拟业务数据、查询函数、金融词典、二级筛选接口模拟。

```text
server/index.js
```

Express 后端、CopilotKit Runtime、模型接入、静态资源托管。

```text
vite.config.js
```

开发环境 Vite proxy，将 `/api/copilotkit` 转发到 `4000`。

## 8. 真实项目迁移建议

迁移到旧 React 项目时，建议按下面顺序：

1. 选择一个试点页面。
2. 梳理页面 state、接口、用户操作。
3. 把页面操作收敛为稳定 action。
4. 用 `useAgentContext` 暴露必要状态。
5. 用 `useFrontendTool` 注册动作。
6. 对二级联动、导出、提交、删除等动作加硬约束。
7. 给每个工具返回结构化结果。
8. 增加审计日志。

## 9. 常见问题

### 对话不响应

检查后端是否启动：

```text
http://localhost:4000/health
```

如果开发环境控制台出现 `ECONNREFUSED`，通常是 `4000` 后端没启动。

### 模型调用失败

检查 `.env` 是否有：

```env
DEEPSEEK_API_KEY=...
```

或者：

```env
OPENAI_API_KEY=...
```

### AI 没有直接筛选二级分类

这是预期行为。

二级分类需要：

1. 金融语义识别。
2. 用户确认。
3. 接口请求候选项。
4. 再设置筛选。

### Docker 访问哪个端口

Docker 模式访问：

```text
http://localhost:4000
```

开发模式访问：

```text
http://localhost:5174
```
